RUN THE VISUALISER

After flashing all 4 boards, here's how to get the live visualiser running.

1 — Position the nodes. Place the 4 boards at the corners of your space, 5–10 metres apart, at waist height (~1m). Node 0 stays plugged into your laptop. The other 3 run on USB power banks. Each OLED shows live signal strength to its peers.

[A] [B] ●──────── 5-10m ────────────● │ │ 5-10m your space 5-10m │ waist height │ │ │ ●──────── 5-10m ────────────● [D] [C]

2 — Install the bridge. You need Python 3 (pre-installed on macOS). Open a terminal and run:

pip3 install pyserial websockets

Then download the project files:

git clone https://github.com/ZeroOneTwo-CreativeTechnologyHub/Liminal.git
cd Liminal

3 — Start the bridge. With Node 0 plugged into USB:

python3 bridge/bridge.py

Leave this running. It reads radio data from Node 0 and sends it to the visualiser.

4 — Open the visualiser. In a new terminal tab:

open visualiser/index.html

Or double-click the file in Finder. The top-left should say LIVE. Now walk between the nodes and watch the mesh respond.

ABOUT

Most LoRa projects treat radio as a transparent pipe. LIMINAL inverts this — the radio link is the data. By watching how signal strength changes across 6 node-to-node paths, the system builds a living map of radio energy flowing through physical space and being disrupted by the bodies within it.

Built at Zero One Two Creative Technology Hub using Heltec WiFi LoRa 32 V3 boards (ESP32-S3 + SX1262). Under £20 per node.
